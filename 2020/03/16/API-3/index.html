API-3'

学习来源

网易云课堂：python全栈工程师微专业：Django章节的API内容

课程目标

  • 构建对网站内容的API：增、删、改、查

    1.创建posts应用，构建相应的API

    1.1配置posts应用

    进入blog/settings.py文件配置INSTALLED_APPS

进入blog/urls.py文件配置posts的api路由（需要先在posts目录下新建urls.py文件）

1.2为posts应用编写模型

进入posts/models.py编写应用的模型

1.3生成样例数据，方便后续测试

进入posts/admin.py文件，把创建的模型类进行注册，增加到管理项目，方便使用admin界面登录后，发布一些博客内容，方便后续测试使用

1.4为posts应用构造api

在posts目录下创建名为api的子目录，并在api子目录下创建serializers.py及views.py文件

  • serializers.py (序列化文件：将对象重构)
    在文件内将要读取的博客文章的内容序列化

  • views.py(api接口的视图文件)

1.5为API配置路由

进入posts的urls.py文件，配置相应的接口路由

2.对构建的API进行测试

测试之前，先打开用户登录的接口进行登录，获取返回的key后，为该测试项目配置Authorization

2.1获取博客文章列表的接口测试

api接口：GET http://127.0.0.1:8000/api/posts/

使用get请求，获取博客文章列表的内容

2.2创建博客文章的接口测试

api接口： POST http://127.0.0.1:8000/api/posts/create/

使用post请求，进行文章创建，需要在Body里提交相应title，content（本次提交以x-www-form-urlencoded方式），在Headers里设置提交的两个字段的value为application/x-www-form-urlencoded，发布成功会返回该文章的详情

2.3获取某一篇博客文章的内容的接口测试

api接口：GET http://127.0.0.1:8000/api/posts/7/

使用get请求，获取指定的某一篇博客文章（即在获取博客文章列表的api后，增加该博客文章的id）

2.4修改某一篇博客文章的内容的接口测试

api接口：PATCH http://127.0.0.1:8000/api/posts/7/

使用patch请求，针对指定的博客文章的api进行提交，需要在Body里提交相应title，content（本次提交以x-www-form-urlencoded方式），在Headers里设置提交的两个字段的value为application/x-www-form-urlencoded，修改成功后会返回文章内容

2.5删除某一篇博客文章的接口测试

api接口：DELETE http://127.0.0.1:8000/api/posts/7/

使用delete请求，针对要删除的博客文章的api进行删除，删除成功后会返回”Successfully deleted”
